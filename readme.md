# 🌍 Wander Mark (WanderingMark)

**Wander Mark** — это комплексная экосистема для экспорта, отображения и управления базами знаний (vaults) из **Obsidian**. Проект превращает ваши сырые Markdown-заметки в полноценное, кроссплатформенное приложение (PWA / Desktop) с поддержкой полнотекстового поиска, графа связей и динамически подгружаемых плагинов.

## ✨ Ключевые возможности

- 🔄 **Полная совместимость с Obsidian**: Поддержка вики-ссылок (`[[ссылка]]`), Frontmatter (YAML), Callouts, и встроенных изображений.
- ⚡ **Быстрый клиент**: Написан на Vue 3 + Vite. Поддерживает работу без интернета (PWA) и может быть скомпилирован как десктопное приложение через Tauri.
- 🧩 **Динамическая система плагинов**: Плагины (ES-модули) компилируются отдельно и могут подгружаться клиентом прямо в runtime по URL! В комплекте уже идут:
  - 🤖 **AI Assistant**: Чат с ИИ на базе ваших промптов (поддержка API AiHubMix).
  - 🇨🇳 **Chinese Dictionary**: Перевод и пиньинь для китайских иероглифов прямо в тексте по клику.
- 🔍 **Умный поиск**: Локальный полнотекстовый поиск по заметкам и тегам с помощью `Fuse.js`.
- 🌗 **Кастомизация и i18n**: Темная/Светлая темы, поддержка нескольких языков (Ru, En, Cn), гибкие настройки отображения контента.
- 🚀 **CLI-утилита**: Автоматизированный парсинг заметок, сборка метаданных (время чтения, связи) и деплой на сервер в одну команду.

## 🏗 Архитектура монорепозитория

Проект организован как монорепозиторий (workspaces) на базе **Bun**:

```text
wander-mark/
├── apps/
│   ├── cli/                        # CLI инструмент для сборки и деплоя заметок из Obsidian
│   ├── client/                     # Frontend приложение (Vue 3, Pinia, Tauri, PWA)
│   ├── server/                     # Простой статический сервер на Bun (для локальной раздачи)
│   ├── plugin-ai-assistant/        # Плагин: ИИ-ассистент
│   └── plugin-chinese-dictionary/  # Плагин: Всплывающий китайский словарь
└── vaults-example/                 # Пример обработанного хранилища (база знаний по корейскому)
```

## 🚀 Быстрый старт

### Требования
- **Bun** версии `>= 1.1.30`

### 1. Установка зависимостей
Выполните команду в корне проекта:
```bash
bun install
```

### 2. Режим разработки (Сборка и запуск)
Запуск всех проектов параллельно (CLI, Клиент, Сервер, Плагины):
```bash
bun run dev
```

Или запуск компонентов по отдельности:
- `bun run dev:client` — запуск фронтенда (Vite, порт 5173)
- `bun run dev:server` — запуск локального файлового сервера (порт 4444)
- `bun --cwd ./apps/cli dev` — запуск сборщика заметок

### 3. Desktop (Tauri)
Если вы хотите запустить проект как нативное десктопное приложение:
```bash
cd apps/client
bun run tauri:dev
```

### 4. Сборка для продакшена
Сборка всех пакетов:
```bash
bun run build
```

## ⚙️ Жизненный цикл (Как это работает)

1. **Создание контента**: Вы пишете заметки в Obsidian (как в примере `vaults-example/Korean`).
2. **Сборка (CLI)**: Утилита `@injurka/wander-mark-cli` парсит папки, резолвит вики-ссылки, удаляет markdown из текста для создания поискового индекса (`search.json`) и собирает карту обратных ссылок (`backlinks.json`). [Подробнее о CLI](./apps/cli/readme.md).
3. **Раздача (Server)**: Статические файлы раздаются вашим сервером (или встроенным `apps/server`).
4. **Отображение (Client)**: Vue-клиент подключается к серверу по URL, скачивает `nav.json` и начинает отрисовывать базу знаний, кэшируя файлы через Service Worker для работы в оффлайне.

## 🧩 Система плагинов (Runtime)

Уникальная фишка **Wander Mark** — это возможность расширять функционал клиента без его пересборки. 

Плагины лежат в `apps/plugin-*`. Они собираются как независимые ES-модули (с `vue` в качестве external-зависимости). Клиент скачивает их по ссылке (или загружает из IndexedDB) и динамически монтирует их компоненты в специальные слоты (`toolbar`, `overlay`, `vault-index` и др.).

### Добавление плагина в клиент:
1. Соберите плагин: `cd apps/plugin-ai-assistant && bun run build`
2. В клиенте откройте меню **"Настройки" -> "Менеджер плагинов"**.
3. Вставьте ссылку на скомпилированный `.js` файл (например: `http://localhost:4444/meta/Korean/plugins/ai-assistant.js`).
